# drag-sort-autoscroll 教程式注释设计

## 背景与目标
- 目标：让“完全小白”通过注释即可理解并能使用 `drag-sort-autoscroll` 里的 4 个示例/逻辑文件。
- 范围：仅补充/调整注释，不改动任何功能逻辑与行为。

## 约束与原则
- 注释中文、短句、分行，强调“做什么/为什么/怎么用”。
- 不解释 TS 语法本身，聚焦拖拽排序场景。
- 术语统一：预览/落点/末尾落点/提交/自动滚动。

## 方案对比
1. 行内讲解
   - 优点：逐行直观。
   - 缺点：代码变长、维护成本高。
2. 顶部教程 + 关键点注释
   - 优点：阅读路径清晰、维护轻。
   - 缺点：需要先读顶部说明。
3. 混合式（采用）
   - 顶部提供步骤教程 + 关键概念清单；关键函数与分支保留行内讲解。
   - 兼顾小白上手与维护成本。

## 设计细节
### 1) useDragSortAutoScroll.ts
- 顶部加入教程区块：用途、适用场景、最小使用步骤、关键概念。
- 函数内重点注释：
  - 预览列表（拖拽中实时重排）。
  - 落点边界计算（上半区/下半区）。
  - drop 前同步最后一次 rAF 结果。
  - 自动滚动容器查找与步长计算。

### 2) useSingleDragSortAutoScroll.ts
- 顶部教程区块与分组版对齐。
- 函数内重点注释：
  - 预览列表与布局比较。
  - 落点边界与末尾落点处理。
  - 自动滚动触发与停止条件。

### 3) MinimalDragSortDemo.vue
- script：说明数据结构、labels 映射用途、composable 返回值用途。
- template：标注分组拖拽区、条目拖拽区、末尾落点区。
- style：仅注释落点虚线/激活态相关样式。

### 4) MinimalSingleDragSortDemo.vue
- script：说明数据结构、`getId` 作用。
- template：标注拖拽区域与末尾落点区。
- style：仅注释落点虚线/激活态相关样式。

## 风险与测试
- 风险：仅注释改动，逻辑风险为 0；主要关注注释是否清晰、术语是否一致。
- 验证：人工审阅 + 运行 `npm run test` 与 `npm run build`（如需）。
