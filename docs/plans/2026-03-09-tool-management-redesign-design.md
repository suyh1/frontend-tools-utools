# 工具管理与主界面重构设计文档（方案 C）

**日期：** 2026-03-09  
**状态：** 已确认  
**范围：** uTools 小窗场景下的空间优化 + 工具组织重构 + 完整管理能力

---

## 1. 背景与问题

当前插件在默认小窗口中存在两个核心问题：

1. 顶部空间占用过高。  
全局壳层头部与工具页内部头部叠加，导致真正工具操作区被明显挤压。

2. 工具选择区组织弱。  
工具数量增长后，单一扁平 Tabs 在视觉和认知上都缺乏层次，查找效率下降。

---

## 2. 目标与非目标

### 2.1 目标

1. 在不牺牲功能的前提下显著释放主操作区高度。
2. 将工具组织升级为“可管理系统”，支持用户自定义分组与排序。
3. 提供完整管理能力：分组管理、拖拽排序、收藏、显示/隐藏、别名、图标、导入导出、恢复默认。
4. 支持“默认配置 + 用户覆盖配置”双层模型，且配置可持久化。

### 2.2 非目标

1. 本阶段不改动各工具核心业务算法。
2. 本阶段不引入新窗口/多进程协同管理。
3. 本阶段不做云同步。

---

## 3. 方案对比与选择

### 3.1 方案 A：轻改（压缩 + 分类）

- 优点：实现快、风险低。
- 缺点：无法满足“用户可管理 + 完整配置能力”。

### 3.2 方案 B：中改（压缩 + 管理基础）

- 优点：结构清晰、可扩展。
- 缺点：仍不覆盖导入导出、别名/图标等完整能力。

### 3.3 方案 C：重构（本次选择）

- 内容：引入独立“工具管理 Drawer”体系，主界面改为紧凑入口 + 收藏条 + 搜索 + 更多工具弹层。
- 结论：满足当前与后续增长诉求，支持“默认 + 用户覆盖”长期演进。

---

## 4. 信息架构

### 4.1 主界面（运行态）

1. 紧凑全局头部（保留）。
2. 收藏工具条（主切换入口）。
3. 搜索框（按名称/关键词检索可见工具）。
4. “更多工具”弹层（分组浏览全部非隐藏工具）。
5. 工具内容区（最大化展示）。

> 决策：去除各工具页面的大型 hero/header，仅保留必要的局部操作文案。

### 4.2 管理界面（配置态）

通过右上角按钮打开 `Drawer`，包含：

1. 分组管理（新增/重命名/删除/排序）。
2. 工具编排（组内排序、跨组移动）。
3. 工具属性（收藏、隐藏、别名、图标）。
4. 配置操作（导入、导出、恢复默认）。

---

## 5. 配置模型设计

## 5.1 默认配置（开发侧）

由代码定义并随版本发布：

1. 默认分组与分组顺序。
2. 默认组内工具顺序。
3. 默认收藏与可见状态。
4. schema 版本号。

### 5.2 用户覆盖配置（运行侧）

记录用户自定义结果：

1. 分组顺序与组内顺序。
2. 工具分组归属。
3. 收藏、隐藏状态。
4. 别名、图标覆盖。
5. `updatedAt` 与 `version`。

### 5.3 合并策略

启动时生成“有效配置”：

`有效配置 = 默认配置 + 用户覆盖`

规则：

1. 用户字段优先。
2. 无效工具 ID 自动忽略。
3. 新增工具自动进入默认分组末尾（避免丢工具）。

---

## 6. 存储策略

按已确认方案：`uTools 为主，localStorage 兜底`

### 6.1 读取顺序

1. 优先读取 uTools 存储。
2. 失败或空值时读取 localStorage。
3. 再失败时回退默认配置。

### 6.2 写入策略

1. 每次配置变更先尝试写 uTools。
2. 无论 uTools 成功与否都写 localStorage（兜底）。
3. 若 uTools 写失败，UI 给出轻提示（不阻断操作）。

---

## 7. 关键交互细节

### 7.1 拖拽排序

1. 分组支持拖拽重排。
2. 工具支持组内重排与跨组拖拽。
3. 拖拽结束后立即更新状态并持久化。

### 7.2 删除分组

必须先选择迁移目标组；若未选则禁止删除，避免工具悬空。

### 7.3 导入导出

1. 导出：下载 JSON（含版本号）。
2. 导入：先校验 schema，再预览变更，再确认应用。
3. 导入失败不覆盖当前配置。

### 7.4 恢复默认

二次确认后清空用户覆盖，回到默认配置。

---

## 8. 异常与回退

1. 存储不可用：仅本次会话生效并提示。
2. 配置损坏：自动回退默认并保留错误日志信息。
3. 工具注册变化：通过 `version + 迁移` 保障兼容。

---

## 9. 测试策略

1. Store 单测：配置合并、迁移、排序、收藏、隐藏、别名、图标。
2. 存储单测：uTools 失败时 localStorage 兜底行为。
3. Shell 组件单测：收藏条、搜索过滤、更多工具弹层、隐藏工具不可见。
4. 管理 Drawer 组件单测：分组增删改、拖拽、导入导出、恢复默认。

---

## 10. 交付与验收标准

1. 小窗中主工具操作区明显增大。
2. 工具组织可按分组与搜索快速定位。
3. 用户可完成完整管理能力并持久化。
4. 导入导出与恢复默认可用且安全。
5. `npm run test` 与 `npm run build` 全通过。
